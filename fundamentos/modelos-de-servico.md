# ☁️ Modelos de Serviço na Cloud

Na computação em nuvem, os serviços são divididos em três modelos principais: IaaS, PaaS e SaaS.

---

# 🏗️ IaaS (Infrastructure as a Service)

## O que é?
É o modelo onde você aluga a infraestrutura básica da nuvem.

---

## O que inclui?
- Máquinas virtuais
- Redes
- Armazenamento
- Sistemas operacionais

---

## Para que serve?
Permite criar e gerenciar servidores completos na nuvem.

---

## Exemplo na AWS
- Amazon EC2
- Amazon VPC

---

## Como funciona?
Você gerencia quase tudo:
- sistema operacional
- aplicações
- dados

A AWS gerencia apenas a infraestrutura física.

---

## Resumo
IaaS = você controla o sistema, a AWS fornece a máquina.

---

# ⚙️ PaaS (Platform as a Service)

## O que é?
É uma plataforma pronta para você desenvolver e executar aplicações.

---

## O que inclui?
- Infraestrutura
- Sistema operacional
- Ambiente de execução
- Ferramentas de desenvolvimento

---

## Para que serve?
Facilitar o desenvolvimento de aplicações sem se preocupar com servidores.

---

## Exemplo na AWS
- AWS Elastic Beanstalk
- AWS Lambda

---

## Como funciona?
Você foca apenas no código.

A AWS cuida do resto:
- servidores
- escalabilidade
- manutenção

---

## Resumo
PaaS = você só programa, a plataforma gerencia tudo.

---

# 📦 SaaS (Software as a Service)

## O que é?
São softwares prontos que você usa diretamente pela internet.

---

## O que inclui?
- Aplicações completas
- Interface pronta
- Hospedagem gerenciada

---

## Para que serve?
Usar sistemas sem precisar instalar ou gerenciar nada.

---

## Exemplo
- Gmail
- Google Drive
- Microsoft 365

---

## Como funciona?
Você apenas acessa e usa.

Tudo é gerenciado pelo provedor.

---

## Resumo
SaaS = você só usa o software.

---

# 🔥 Comparação rápida

| Modelo | Controle do usuário | Exemplo |
|--------|--------------------|---------|
| IaaS   | Alto               | EC2     |
| PaaS   | Médio              | Lambda  |
| SaaS   | Baixo              | Gmail   |

---

# 🎯 Resumo final

- IaaS → você gerencia tudo
- PaaS → você gerencia só o código
- SaaS → você só usa o software
